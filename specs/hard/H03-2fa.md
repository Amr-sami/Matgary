## Why

Owner credential compromise = full tenant takeover. Single weakest link in the auth model today. Pen-tester (E2) will flag absence. Pair with H09 (session revocation) so a compromised user can recover.

## Acceptance criteria

- [ ] Migration adds `users.totp_secret text`, `users.totp_enabled_at timestamptz`, `users.recovery_codes_hash text[]`.
- [ ] `/account/security` enrollment flow: QR code from `otpauth://totp/Matgary:<email>?secret=...&issuer=Matgary`, verify a 6-digit code from the user's app to commit; on commit, display 8 one-time recovery codes once (downloadable as `.txt`) and never again.
- [ ] Login `authorize` in `lib/auth.ts`: after password success, if `totp_enabled_at` is set, returns a "needs 2FA" partial credential that the UI uses to prompt for a 6-digit code or a recovery code on a follow-up `POST /api/auth/2fa/verify`.
- [ ] Recovery code consumed on use: removed from `recovery_codes_hash`. UI shows the remaining count.
- [ ] Regen flow: "regenerate recovery codes" requires current password + a valid TOTP.
- [ ] Disable 2FA: requires current password + valid TOTP.
- [ ] Rate-limit `auth.totp` 5 / 15 min / user-id; recovery code attempts share the bucket.
- [ ] Activity log: `account.2fa_enable`, `account.2fa_disable`, `account.recovery_code_used`, `account.recovery_codes_regenerated`.
- [ ] Owner role only for v1 â€” staff enrollment hidden behind a feature flag check.
- [ ] Unit tests: TOTP verify accepts the previous + current + next 30 s window; rejects window-2; recovery code one-shot; rate limit consumption.

## Implementation plan

1. Add `otplib` (smaller surface than `speakeasy`).
2. Generate `totp_secret` server-side, 20 random bytes, base32.
3. Recovery codes: 8 Ã— `crypto.randomBytes(5).toString("hex")` (10 hex chars each, grouped `XXXXX-XXXXX`), bcrypt-hashed individually.
4. Login flow split: `authorize` checks password, sets `mfaPending: true` in returned object if 2FA enrolled. JWT callback short-circuits and refuses to issue a session-id JWT until `/api/auth/2fa/verify` succeeds and clears the pending flag.
5. UI: new `/login/2fa` page with code input + "use a recovery code" toggle.
